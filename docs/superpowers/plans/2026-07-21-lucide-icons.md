# Lucide Icons Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the four text/emoji glyphs standing in for icons (nav hamburger, Galerie external-link, admin delete/edit buttons) with real Lucide SVG icons, and document the resulting icon usage convention.

**Architecture:** Vendor Lucide's UMD build as a static file (curl, no devDependency), load it site-wide via `footer.php`, and swap each glyph for a `<i data-lucide="...">` placeholder that `lucide.createIcons()` converts to inline `<svg>` — called once globally on page load (`main.js`) and again after the admin event list is rebuilt dynamically (`planning_repet.js`).

**Tech Stack:** Vanilla JS/CSS (no bundler), PHP 8.4 partials, Lucide 1.25.0 (vendored).

## Global Constraints

- No new npm devDependency — vendor by curling the static file, matching the existing `app/assets/vendor/` convention (`i18next.min.js`, `bulma.min.css`).
- Lucide version: **1.25.0**. File: `app/assets/vendor/lucide.min.js`. Expected size **411977 bytes**, sha256 `89678151bc9de869a48a8b430331073cb359478146b34dc103440f18bf143549`.
- Icon markup/init mechanism: `<i data-lucide="icon-name"></i>`, converted by `lucide.createIcons()`.
- Standard icon size app-wide: **24×24px**. No per-spot ad hoc sizing.
- Style is outline/stroke-only (`fill="none"`, `stroke="currentColor"`) — never set `fill` or `stroke` directly on an icon; color comes from the parent element's CSS `color`.
- Exact mapping: hamburger → `menu`, Galerie link → `external-link`, delete → `trash-2`, edit → `pencil`.
- Run `npm run check` before considering the work done (phpcs, PHPUnit, eslint, stylelint, prettier, secret guard).
- Never hand-edit `public/` — it's generated by `npm run build` from `app/`.

### Task 1: Vendor `lucide.min.js` + document it

**Files:**
- Create: `app/assets/vendor/lucide.min.js`
- Modify: `app/assets/vendor/README.md`

**Interfaces:**
- Produces: a global `lucide` object (from the UMD build) with a `lucide.createIcons()` function, available to any script loaded after this file on the page. Later tasks load this via a `<script>` tag (Task 2) and call `lucide.createIcons()` (Tasks 2 and 4).

- [ ] **Step 1: Download the file**

The canonical source is jsdelivr, matching the existing vendored files. If jsdelivr is unreachable (some sandboxed environments block arbitrary CDN hosts while allowing `registry.npmjs.org`), extract the identical file from the npm tarball instead — both produce byte-identical output. Try jsdelivr first:

```bash
curl -fL "https://cdn.jsdelivr.net/npm/lucide@1.25.0/dist/umd/lucide.min.js" \
  -o app/assets/vendor/lucide.min.js
```

If that fails (connection/proxy error), use the npm-registry fallback:

```bash
curl -fL "https://registry.npmjs.org/lucide/-/lucide-1.25.0.tgz" -o /tmp/lucide.tgz
tar -xzf /tmp/lucide.tgz -C /tmp package/dist/umd/lucide.min.js
cp /tmp/package/dist/umd/lucide.min.js app/assets/vendor/lucide.min.js
rm -rf /tmp/lucide.tgz /tmp/package
```

- [ ] **Step 2: Verify the downloaded file**

```bash
wc -c app/assets/vendor/lucide.min.js
sha256sum app/assets/vendor/lucide.min.js
head -c 120 app/assets/vendor/lucide.min.js
```

Expected: `411977 app/assets/vendor/lucide.min.js`, sha256
`89678151bc9de869a48a8b430331073cb359478146b34dc103440f18bf143549`, and the
head output starts with:

```
/**

 * This source code is licensed under the ISC license.
```

If the size/hash don't match exactly, stop and re-check the download — do

### Task 2: Load Lucide site-wide and initialize it

**Files:**
- Modify: `app/partials/footer.php`
- Modify: `app/assets/js/main.js`
- Modify: `eslint.config.js`

Task 3 and Task 4 rely on this running automatically — they only need to add `data-lucide` markup, not call `createIcons()` themselves for the initial page render.

Replace with:

- [ ] **Step 2: Declare the `lucide` global for ESLint**

In `eslint.config.js`, add a new block after the existing `i18next.js`-specific
block (the last block in the file). Match the file's existing comment style:

```js
  {
    // `lucide` (vendored, app/assets/vendor/lucide.min.js) is a shared
    // global consumed directly by main.js (global icon init) and
    // planning_repet.js (re-init after the admin event list is rebuilt).

    files: ['app/assets/js/main.js', 'app/assets/js/planning_repet.js'],
```
